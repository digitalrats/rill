* Общая концепция

** Видение
Kama Audio — это модульный open-source фреймворк для создания аудиоприложений на Rust,
а также экосистема совместимых продуктов. Ключевая идея: минимальное стабильное ядро
определяет базовые абстракции, а вся функциональность реализуется в специализированных
крейтах. На базе фреймворка строятся конкретные продукты, первым из которых станет
Drift — сервер эффектов для live coding сообщества.

** Целевая аудитория (alpha)
- Сообщества live coding: SuperCollider, TidalCycles, FoxDot
- Разработчики аудио на Rust, интересующиеся модульными системами
- Энтузиасты, создающие собственные инструменты и эффекты

** Ключевые принципы
- Минимальное ядро: только базовые абстракции, всё остальное — в крейтах
- Модульность: каждый крейт имеет чёткую ответственность
- Открытость: публичный API, документированный протокол
- Производительность: zero-cost abstractions, real-time safety
- Расширяемость: реестр фабрик узлов, возможность динамической загрузки
- Совместимость: единый протокол для взаимодействия компонентов экосистемы

** Фундаментальные архитектурные решения
- =TimeProvider= — единый источник времени для всех компонентов
- =SignalBus= — гибкая шина событий с настраиваемыми каналами
- Реестр фабрик узлов (=NodeFactory=) — создание узлов по имени
- DSP как эффект в контексте (=DspContext=) — функции, получающие время, параметры, сигналы, буферы
- Разделение на цифровые и аналоговые крейты для ясности происхождения алгоритмов
- Клиент-серверная модель с единым протоколом управления (Kama Protocol)

- Все крейты зависят от =kama-core= (минимальное ядро)
- DSP-крейты зависят от =kama-dsp-common=, =kama-buffers=, =kama-wdf=

** Ключевые абстракции

*** TimeProvider
#+BEGIN_SRC rust
pub trait TimeProvider: Send + Sync {
    fn sample_rate(&self) -> f64;
    fn position_samples(&self) -> u64;
    fn position_seconds(&self) -> f64 { ... }
    fn position_beats(&self) -> f64 { ... }
    fn bpm(&self) -> f64;
    fn set_bpm(&self, bpm: f64);
    fn tick_info(&self) -> TickInfo;
}

pub struct TickInfo {
    pub bar: u32,
    pub beat: u8,
    pub sixteenth: u8,
    pub sample_pos: u64,
}
#+END_SRC

*** SignalBus
#+BEGIN_SRC rust
pub trait Signal: Send + 'static {}

pub struct SignalBus<T: Signal> {
    tx: Sender<T>,
    rx: Receiver<T>,
    config: BusConfig,
}

pub enum BusConfig {
    Bounded(usize, OverflowPolicy),
    Unbounded,
}

pub enum OverflowPolicy { DropOldest, DropNewest, Block }
#+END_SRC

*** DspContext
#+BEGIN_SRC rust
pub struct DspContext<'a> {
    pub time: &'a dyn TimeProvider,
    pub params: &'a ParameterMap,
    pub signals: &'a dyn SignalBus,
    pub buffers: &'a BufferRegistry,
    pub sample_rate: f32,
    pub block_size: usize,
    pub block_position: usize,
}
#+END_SRC

*** NodeFactory и реестр
#+BEGIN_SRC rust
pub trait NodeFactory: Send + Sync {
    fn create(&self) -> Box<dyn AudioNode>;
    fn metadata(&self) -> NodeMetadata;
}

lazy_static! {
    static ref NODE_REGISTRY: RwLock<HashMap<String, Box<dyn NodeFactory>>> = Default::default();
}
#+END_SRC

** Протокол взаимодействия (Kama Protocol)
- Основа: OSC для команд, CBOR для больших данных
- Базовые команды:
  - =/service/register=, =/service/discover= — обнаружение сервисов
  - =/transport/start=, =/stop=, =/bpm=, =/position= — транспорт
  - =/node/add=, =/node/remove=, =/node/set= — управление узлами
  - =/connect=, =/disconnect= — соединения
  - =/graph/get=, =/graph/share= — обмен графами
  - =/parameter/set=, =/parameter/get= — параметры (альтернатива /node/set)
- Discovery: UDP broadcast / mDNS
